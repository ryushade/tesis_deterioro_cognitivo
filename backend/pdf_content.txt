Descripción del producto
La plataforma de Inteligencia Artificial (en adelante, IA) para la Detección Tem-
prana de la Enfermedad de Alzheimer a través de la Voz (IAEAV) es una solución
tecnológica orientada al cribado cognitivo remoto mediante el análisis automático de
muestras de voz. Su desarrollo responde a la necesidad de disponer de herramientas
escalables, accesibles y no invasivas que permitan apoyar la detección temprana del
deterioro cognitivo, reduciendo la dependencia de evaluaciones exclusivamente pre-
senciales y de recursos clínicos altamente especializados.
En cuanto a las partes de la plataforma, ésta integra una aplicación móvil guiada pa-
ra la captura estructurada de tareas de habla, una infraestructura segura de gestión de
datos. Además, inlcuye un sistema de procesamiento asíncrono para la inferencia me-
diante modelos de aprendizaje profundo. Este enfoque permite combinar la facilidad
de uso para pacientes y profesionales con requisitos estrictos de seguridad, trazabili-
dad y reproducibilidad, aspectos especialmente relevantes en entornos biomédicos y
clínicos.
En su conjunto, IAEAV se concibe como un ecosistema integral en el que la voz
actúa como biomarcador digital no invasivo y de bajo coste. A partir de ella, la plata-
forma analiza información textual, acústica y multimodal para apoyar la identificación
de patrones asociados a la enfermedad de Alzheimer. Entre sus principales aportacio-
nes destacan el empleo de técnicas de IA Explicable (en adelante, XAI, por sus siglas en
inglés eXplainable Artificial Intelligence), la validación de arquitecturas avanzadas para
clasificación automática y la consolidación de un corpus clínico en español específica-
mente diseñado para este problema.
La plataforma ha sido diseñada para dar servicio tanto a equipos de investigación
como a profesionales sanitarios, facilitando escenarios de telemonitorización y evalua-
ción accesible. Del mismo modo, aporta beneficios directos a los pacientes, al permitir
evaluaciones menos intrusivas, más cómodas y potencialmente más frecuentes, favo-
reciendo así el seguimiento longitudinal y la detección precoz.

Funcionalidades
La plataforma IAEAV proporciona un conjunto de herramientas orientadas a la cap-
tura, protección, procesamiento y análisis de muestras de voz para el cribado temprano
de la enfermedad del Alzheimer. Sus funcionalidades principales incluyen:
1. Captura guiada de datos de voz:
Registro de tareas de habla desde una aplicación móvil.
Guiado del usuario durante el proceso de grabación.
Validaciones acústicas locales para verificar la calidad de la muestra.
Envío tolerante a fallos mediante políticas de reintento ante redes inestables.
2. Gestión segura e interoperable de la información
Exposición de una API REST con separación entre endpoints y lógica de
negocio.
Control de acceso basado en token.
Validación de metadatos asociados a cada grabación.
Registro exhaustivo de eventos para asegurar trazabilidad.
3. Protección criptográfica de audios y datos sensibles
Cifrado local del audio mediante clave simétrica AES-GCM.
Cifrado posterior de la clave simétrica con la clave pública del servidor me-
diante RSA-OAEP.
Recuperación segura de la clave únicamente por parte del servidor.
Garantía de confidencialidad e integridad tanto en tránsito como en almace-
namiento.
4. Procesamiento asíncrono para inferencia con IA
Ejecución de tareas pesadas en segundo plano.
Control automático de calidad, por ejemplo mediante relación señal-ruido.
Activación desacoplada de los modelos de inferencia profunda.
Posibilidad de escalado horizontal dinámico sin bloquear la API principal.

Código fuente
A continuación se describe la estructura de los repositorios de código fuente que
componen la plataforma IAEAV.
Carpeta iaeav_backend:
El backend es una API REST construida sobre Flask 2.3 y Gunicorn, con PostgreSQL
15 como base de datos, Redis 7 como broker de tareas asíncronas (Celery) y cifrado
de extremo a extremo basado en RSA-OAEP-256 y AES-256-GCM. El repositorio se
organiza de la siguiente manera:
factory.py: fábrica de la aplicación Flask (Application Factory pattern); inicializa
las extensiones (SQLAlchemy, Flask-Migrate, CORS) y registra todos los Blue-
prints de la API.
models.py: modelos SQLAlchemy que definen el esquema de la base de datos
PostgreSQL: User (con rol, hash de contraseña y proveedor OAuth), Recording
(grabaciones cifradas con su estado de análisis, clase predicha y transcripción)
y UploadSession (sesiones de subida por fragmentos con control de integridad
SHA-256).
celery_app.py: configuración de Celery con Redis como broker; define la cola de
tareas asíncronas para el procesamiento de audio.
wsgi.py: punto de entrada WSGI para Gunicorn en producción (4 workers por
defecto).
manage.py: comandos CLI de Flask personalizados (flaskcreate-admin, inte-
gración con Flask-Migrate para migraciones de base de datos).
api/: Blueprints REST organizados por dominio funcional, cada uno con su pro-
pia ruta base bajo /api/v1/:
• auth.py: registro de usuario, login (devuelve JWT RS256), logout e invali-
dación de token, y flujo completo OAuth/OIDC con Google.
• users.py: CRUD completo de usuarios (crear, listar, actualizar, eliminar),
gestión de roles RBAC y consulta del perfil propio (GET/PUT/users/me).

• snr.py: cálculo de la relación señal-ruido (SNR) mediante detección de acti-

de silencio al inicio y al final de la grabación.

(clase predicha, confianza, transcripción) en PostgreSQL.

Carpeta iaeav_frontend:
El frontend es una aplicación web de página única (Single Page Application) cons-
truida con React 19, TypeScript 5.9 en modo estricto y Vite 7 como build tool. Implementa
autenticación JWT y un panel de control con contenido adaptado al rol del usuario. La
estructura del repositorio es la siguiente:
App.tsx: componente raíz que define el árbol de rutas con React Router 7; protege
las rutas privadas redirigiendo al login si no existe sesión activa.
main.tsx: punto de entrada de la aplicación; monta el BrowserRouter y el Auth
Provider sobre el árbol de componentes.
contexts/AuthContext.tsx: contexto global de autenticación; almacena el JWT en
localStorage, lo decodifica para extraer el nombre de usuario, rol y email, expo-
ne las funciones de login y logout, y gestiona la expiración automática del token
(1 hora).
services/apiService.ts: capa de comunicación con la API REST del backend; cen-
traliza todas las llamadas HTTP añadiendo automáticamente la cabecera Author
ization:Bearer<token> y redirigiendo al login ante un error 401.
pages/: vistas principales de la aplicación:
• LoginPage.tsx: formulario de login con credenciales locales y botón de login
con Google OAuth mediante redirección al backend.
• RegisterPage.tsx: formulario de registro de nuevo usuario con validación en
cliente.
• DashboardPage.tsx: panel principal con pestañas cuyo contenido varía se-
gún el rol del usuario autenticado (ver tabla de funcionalidades por rol en
el capítulo anterior).
• ProfilePage.tsx: edición del perfil personal (nombre, email) y cambio de
contraseña.
components/: componentes reutilizables organizados por función:
• FileUploadForm.tsx: formulario de subida de archivos con barra de progre-
so en tiempo real y validación del tipo de archivo en cliente.
• AdminUploadsTable.tsx: tabla de archivos con permisos CRUD completos
(admin y manager); permite ver todos los archivos, descargarlos y eliminar-
los.
• OperatorUploadsTable.tsx: tabla de archivos en modo solo lectura para el
rol operador.

Carpeta ia_multimodal:
Este módulo contiene el código para el entrenamiento y evaluación de un modelo
multimodal orientado a la detección de Alzheimer y a la estimación de deterioro cogni-
tivo a partir de información lingüística y acústica derivada de la voz. A diferencia de un
sistema puramente basado en audio o en texto, este entorno combina transcripciones
automáticas con embeddings acústicos extraídos mediante modelos ASR, permitien-
do estudiar de forma conjunta el contenido verbal y las características temporales del
audio.
El flujo general del codigo parte de archivos de audio, que son procesados median-
te reconocimiento automático del habla para generar, por un lado, transcripciones tex-
tuales y, por otro, representaciones acústicas. A partir de estas dos fuentes, el sistema
entrena el modelo y lo evalua.
La estructura del codigo es la siguiente:
classify_train.py: script principal de entrenamiento para la tarea de clasificación.
Implementa un modelo multimodal que combina embeddings textuales obteni-
dos con BERT y embeddings acústicos, realizando entrenamiento y evaluación
mediante validación cruzada estratificada. Incluye cálculo de métricas, selección
del mejor modelo por fold y almacenamiento de resultados y configuraciones de
entrenamiento.

mmse_train.py: script de entrenamiento para la tarea de regresión de la puntua-
ción MMSE. Utiliza la arquitectura multimodal para predecir una variable con-
tinua asociada al estado cognitivo, optimizando con pérdida MSE y evaluando
mediante RMSE.
audio_asr_to_text.py: utilidades de preprocesamiento que transforman audios
en transcripciones automáticas y embeddings acústicos. Soporta distintos flujos
de trabajo, que posteriormente son utilizados por el modelo.
recognize.py: implementación de los componentes de reconocimiento automá-

entrada acústica del sistema multimodal.
check_pause_length.py: script de análisis exploratorio sobre pausas y patrones

cialmente relevantes para la caracterización del habla.
dataloaders.py: contiene las implementaciones de los conjuntos de datos utiliza-

layers.py: define diferentes capas empleadas en el modelo multimodal.
core.py: funciones auxiliares de procesamiento textual. Implementa la carga y

helpers.py: utilidades generales del proyecto, principalmente para la gestión del

CTC). La estructura del repositorio es la siguiente:
src/api/app.py: API principal con dos endpoints: GET/health (comprueba que
el modelo está cargado y listo para recibir peticiones) y POST/predict (ejecuta el

Carpeta ia_audio:
Este módulo contiene el código de investigación para el desarrollo y evaluación
de nuevos modelos de inteligencia artificial orientados a la detección de indicadores
de Alzheimer a partir de señales de voz. A diferencia del sistema en producción, este
entorno permite experimentar con distintas configuraciones de modelos, representa-
ciones espectrales y estrategias de entrenamiento de forma reproducible.
El flujo completo implementado parte de señales de audio en formato WAV, que son
procesadas dinámicamente durante el entrenamiento. En lugar de utilizar espectrogra-
mas precomputados, el modelo trabaja directamente con el audio completo, generando
ventanas temporales y sus correspondientes representaciones espectrales dentro de la
propia red neuronal, lo que permite una mayor flexibilidad y eficiencia experimental.
La estructura del repositorio es la siguiente:
train.py: script principal de entrenamiento que implementa todo el pipeline ex-
perimental: carga de datos, construcción del modelo, entrenamiento con soporte
para mixed precision, validación, test y evaluación mediante múltiples métricas
(accuracy, recall, F1, AUC, entre otras). Además, gestiona la ejecución mediante
validación cruzada estratificada y el almacenamiento de resultados, históricos y
predicciones.
datasets/: contiene las implementaciones de los conjuntos de datos utilizados:
• iaeav_dataset.py: define el dataset principal, que carga archivos WAV com-
pletos, realiza re-muestreo, conversión a mono y construye particiones es-
tratificadas en train/val/test mediante Stratified K-Fold.
• iaeav_dataset_balanced.py: variante que implementa balanceo de clases me-
diante undersampling en el conjunto de entrenamiento, manteniendo la dis-
tribución original en validación y test.
models/: implementaciones de los modelos de deep learning:
• resnext_audio.py: adaptación de arquitecturas ResNeXt para el procesamien-
to de espectrogramas, incluyendo mecanismos de preservación temporal y
atención en el eje temporal.

Instalación paso a paso
A continuación se describe el proceso completo para desplegar la plataforma IAEAV
en un entorno local o de producción, partiendo desde cero.
1. Instalar Docker.
Descargar e instalar Docker Desktop desde https://docs.docker.com/get-d
ocker/ (disponible para Windows, macOS y Linux). En servidores Linux sin
interfaz gráfica se puede instalar únicamente Docker Engine y el plugin Docker
Compose V2.
Verificar que la instalación es correcta ejecutando docker-{}-version y docker
composeversion en un terminal. Se requiere Docker Compose V2 (el comando
es dockercompose, sin guion).
2. Configurar variables de entorno.
Copiar la plantilla de variables del backend con cpbackend/deploy/.env.ex
amplebackend/deploy/.env y editar el fichero resultante con las credenciales
reales: contraseña de PostgreSQL (POSTGRES\_PASSWORD), contraseña de Redis
(REDIS\_PASSWORD), clave secreta de Flask (SECRET\_KEY) y contraseña del
administrador inicial (ADMIN\_PASSWORD).
Generar las cuatro claves criptográficas ejecutando bashbackend/deploy/scr
ipts/gen-keys.sh desde la raíz del repositorio del backend. El script crea los
ficheros jwt\_priv.pem, jwt\_pub.pem, upload\_priv.pem y upload\_pub.pem
en backend/deploy/keys/; estas claves son las que usa el backend para firmar
los JWT y para el cifrado de sobre de los audios.
3. Arrancar servicio de IA.
Descargar los pesos del modelo (aproximadamente 12 GB en total) ejecutando cd
ai-inference/\&\&./download-weights.sh. El script obtiene automáticamente,
en este orden: el clasificador WERT fine-tuned desde la carpeta del proyecto, el
modelo Wav2Vec2 para inglés desde el repositorio fairseq de Meta AI, el modelo
